1. Introduction & Objectives
This group project is the main practical component of this course. The primary goal is to design, develop,
and deploy a fully functional mobile application that solves a real-world problem. The application must
leverage modern cloud computing services for its backend infrastructure and business logic.

Learning Objectives:
●
●
●
●
●

To gain practical experience in the complete mobile application development lifecycle.
To design and implement a scalable, cloud-native application architecture.
To integrate and manage multiple third-party cloud computing services (PaaS/SaaS/IaaS).
To design, build, and deploy a custom "in-house" web service (API) to support the mobile
application.
To practice project management, teamwork, and technical documentation skills.

2. Project Task
Your group is tasked to propose, design, develop, and document a mobile application for either Android
or iOS (or cross-platform using frameworks like Flutter or React Native).
The core of this project lies in the architecture. The application must not be a simple standalone app. It
must be a cloud-integrated solution that meets the specific technical requirements below.

3. Technical Requirements

requirements:

a. Mobile Application (Android/iOS)
You must develop a native or cross-platform mobile application. Your group will define the problem
statement, target audience, and key features of the app.

4. Recommended Methodology
While you are free to choose your project management methodology, we strongly recommend an Agile
approach (like Scrum). This involves working in "sprints" (e.g., 1-2 week cycles) to complete specific
features.

Suggested Project Phases:
1. Phase 1: Planning & Design (Weeks 1-3)
○ Ideation: Define the app's problem statement, target users, and core features.
○ Tech Stack: Decide on your mobile framework (Native, Flutter, React Native), your 3+ thirdparty services, and your in-house API tech.
○ Architecture: Create a detailed system architecture diagram showing how the mobile app,
third-party services, and your in-house API connect.
○ Design:
■ Create UI/UX wireframes and mockups (use tools like Figma or Balsamiq).
■ Design your database schema (e.g., your Firestore collections or SQL tables).
■ Define the API contract (endpoints, request/response formats) for your in-house web
service.
2. Phase 2: Development & Integration (Weeks 4-10)
○ Set up your cloud projects (Firebase, AWS, etc.) and development environments.
○ Backend First: Develop and deploy the first version of your in-house web service.
○ Mobile App: Begin building the mobile app UI.
○ Integrate:
■ Connect the app to cloud authentication.
■ Connect the app to the cloud database and storage.
■ Integrate your in-house web service.
■ Integrate all other cloud services.
○ Iterate: Work in sprints, testing features as you build them.
3. Phase 3: Testing & Deployment (Weeks 11-12)
○ Perform thorough testing: unit testing, integration testing, and user acceptance testing (UAT).
○ Fix bugs and refine the UI/UX.
○ Ensure your in-house web service is stable and deployed.
○ Prepare the final mobile application build (e.g., APK file).
4. Phase 4: Documentation & Presentation (Weeks 13-14)
○ Write the final project report (see structure below).
○ Record and edit your video presentation.
